# Decision 031: Final BRD v3 is a shared team-authored document

Date: `2026-04-17`

## Status
Accepted - approved on `2026-04-17`.

## Context
- The midterm package used a member-wise split where each team member owned `2` UCs.
- That split was useful for the midterm submission workflow, but it does not match how the team wants to produce the final BRD.
- The user explicitly clarified that `BRD final v3` should be written jointly by the team and should no longer be treated as a collection of per-member BRD slices.

## Decision
For the final-project lane:

- `BRD final v3` is a **shared team-authored document**
- the old midterm `8 members x 2 UCs` allocation is retained only as historical context
- final BRD writing must not assume fixed per-member ownership blocks
- the local final BRD UC lane may still preserve the existing `UC-01..UC-16` IDs, but those IDs are no longer tied to individual member ownership

## Consequences
- Midterm assignment tables in the current UC specification package should be treated as archival context only.
- Final BRD planning, outlining, and writing should be organized around one coherent document, not around member-by-member partitions.
- Repo status, scope, and guidance docs should state this explicitly so later sessions do not drift back into the midterm workflow model.
