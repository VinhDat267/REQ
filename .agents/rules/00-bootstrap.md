# Bootstrap

- Before substantial work, read `AGENTS.md`, `.agents/context/current-status.md`, and `.agents/context/project-overview.md`.
- Treat `AGENTS.md` as the shared cross-tool foundation for project workflow and operating rules.
- Keep root markdown requirement files as the source of truth; `.agents/` is maintained project memory and workspace customization, not a second source of truth.
- When rules conflict, prefer direct user instructions first, then `GEMINI.md`, then `AGENTS.md`, then `.agents/context/*`, `.agents/decisions/*`, `.agents/specs/*`, `.agents/rules.md`, and `.agents/rules/*`.
- Use `.agents/workflows/*` as reusable workflows on demand, not as always-on rule files.
